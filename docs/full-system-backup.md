# Nova ERP - الموسوعة البرمجية والتوثيقية الشاملة (نسخة احتياطية كاملة)

هذا المستند هو المرجع النهائي لنظام Nova ERP. تم إعداده ليكون "خزنة برمجية" تحتوي على المنطق، الهيكل، والأكواد المصدرية لكل جزء في النظام.

- تفرض صلاحيات الأدوار (Admin, Accountant, Engineer).

## 💾 القسم 3: طبقة البيانات (Data Layer)

### 3.1 - التعريفات الرئيسية (src/lib/types.ts)
هذا هو "القاموس" الذي يفهمه النظام:

```typescript
// العميل: يمثل ملف العميل الرئيسي
export interface Client {
    id: string;           // المعرف الفريد من Firebase
    fileId: string;       // رقم الملف (مثال: 1/2024)
    nameAr: string;       // الاسم بالعربي - أساس البحث والتقارير
    assignedEngineer?: string; // ربط الموظف المسؤول
    status: 'new' | 'contracted' | 'cancelled'; // دورة حياة العميل
}

// القيد المحاسبي: العمود الفقري للمالية
export interface JournalEntry {
    entryNumber: string;  // رقم تسلسلي (JV-2024-0001)
    date: Timestamp;      // تاريخ القيد الفعلي
    totalDebit: number;   // إجمالي المدين (يجب أن يساوي الدائن)
    lines: {
        accountId: string; // الحساب المتأثر من الشجرة
        debit: number;
        credit: number;
        auto_profit_center?: string; // ربط القيد بالمشروع (مركز ربحية)
    }[];
}
```

## 💰 القسم 4: المنطق المحاسبي (Accounting Logic)

### 4.1 - شجرة الحسابات (COA)
تعتمد ترميزاً خماسياً:
- **1**: الأصول (1101: نقدية، 1102: عملاء).
- **2**: الالتزامات (2101: موردون).
- **4**: الإيرادات (4101: إيرادات عقود).
- **5**: التكاليف (51xx: تكاليف مباشرة، 52xx: مصاريف إدارية).

### 4.2 - القيود التلقائية (Auto-Chains)
1. **اعتماد عقد**: يولد قيد (مدين: العميل / دائن: إيرادات غير محققة).
2. **سند قبض**: يولد قيد (مدين: البنك / دائن: العميل).
3. **استلام بضاعة**: يولد قيد (مدين: المخزون / دائن: المورد).

## 🏗️ القسم 5: موديول المقاولات والـ WBS

### 5.1 - محرك الـ BOQ
يسمح ببناء شجرة بنود غير منتهية. يتم حساب التكلفة التقديرية مقابل الفعلية (من واقع المشتريات) لاستخراج "الانحراف المالي" لكل بند.

### 5.2 - الزيارات الميدانية والـ GPS
عند ضغط "تأكيد إنجاز" في الموقع:
1. يتم التقاط إحداثيات المهندس.
2. يتم تحديث مرحلة العمل في المشروع.
3. يتم التحقق من "شرط الدفعة" في العقد؛ إذا تطابق، يولد **مستخلصاً** وقيداً محاسبياً آلياً.
