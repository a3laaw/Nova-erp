# Nova ERP - الموسوعة البرمجية والتوثيقية النهائية (نسخة الحماية والرقابة الكاملة)

هذا المستند هو "الصندوق الأسود" لنظام Nova ERP. يحتوي على شرح تفصيلي للمنطق البرمجي، هيكل البيانات، وخرائط العلاقات المعقدة، ليكون المرجع المطلق للنسخ الاحتياطي وفهم "عقل النظام".

---

## 🏗️ القسم 1: المعمارية الهيكلية (System Architecture)

يعتمد النظام على Next.js 14 (App Router) مع Firebase كمحرك بيانات لحظي. التفاعل بين "الميدان" (موقع المشروع) و"المكتب" (المحاسبة) يتم عبر **أتمتة القيود**.

```text
src/
├── ai/                     # محركات الذكاء الاصطناعي (Gemini 1.5 Flash)
├── app/                    # المسارات الذكية (App Router)
│   ├── dashboard/          # مركز العمليات (محاسبة، مقاولات، موارد بشرية)
├── components/             # المكونات الرسومية (UI)
├── services/               # العقل المنطقي (الرواتب، التواريخ، الإشعارات)
├── lib/                    # الأنماط (Types) والأدوات (Utils)
└── hooks/                  # خطافات جلب البيانات التحليلية اللحظية
```

## 💾 القسم 3: طبقة البيانات (Data Schema)

*   **العميل (Client)**: الجذر الرئيسي، لا حركات بدون ملف عميل.
*   **المعاملة (Transaction)**: تمثل الخدمة، وهي مركز الربحية الأساسي.
*   **القيد (JournalEntry)**: الضابط المالي، يضمن توازن ميزان المراجعة.
*   **المشروع (Project)**: الكيان التنفيذي المرتبط بجدول كميات (BOQ).

## 💰 القسم 9: المنطق المحاسبي والتكاليف (Accounting Engine)

### 9.1 - سلاسل التفاعلات التلقائية (The Smart Chains)
1.  **اعتماد العقد** ➔ إنشاء قيد مديونية للعميل.
2.  **استلام بضاعة (GRN)** ➔ زيادة المخزون وإثبات مديونية المورد.
    - **قاعدة ذهبية**: يعتمد النظام "مبلغ الاستلام الفعلي" في كافة التقارير المالية للمشروع لضمان دقة الربحية.
3.  **تأكيد زيارة ميدانية** ➔ تحديث الـ WBS ➔ اقتراح مستخلص آلي.

## 📦 القسم 11: الرقابة المخزنية المتقدمة (Triple Shield)

### 11.1 - فحص الرصيد اللحظي (Stock Shield)
يمنع النظام إجراء أي "مردود مشتريات" أو "تسوية تلف" بكمية أكبر مما هو متوفر فعلياً في المستودع المختار.

### 11.2 - الرقابة المتقاطعة للموردين (Virtual Vendor Warehouse)
يمنع النظام إرجاع بضاعة للمورد "أ" إذا كان قد ورّد كمية أقل، حتى لو كان المخزن العام يحتوي على كميات وافرة من موردين آخرين.

### 11.3 - درع مبيعات العميل (Client Sales Shield)
يمنع النظام استلام مرتجعات من العميل بكمية أكبر مما اشتراه فعلياً في سجلات مبيعاته السابقة.
