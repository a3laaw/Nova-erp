# نظرة شاملة على أكواد ونظام Nova ERP

هذا المستند هو المرجع الموحد (Single Source of Truth) لكافة الأكواد والمنطق البرمجي في النظام. تم تجميعه لتوفير "نسخة احتياطية معرفية" تمكنك من فهم وإعادة بناء النظام في أي وقت.

---

## 🏗️ 1. الهيكل العام والفلسفة البرمجية (System Architecture)

يعتمد النظام على فلسفة **"تكامل البيانات اللحظي"**؛ أي أن الإجراء الإداري أو الميداني يولد أثراً مالياً ومخزنياً فورياً.

### التقنيات الأساسية:
- **Frontend**: Next.js 14 (App Router) + TypeScript.
- **Styling**: Tailwind CSS + ShadCN UI (تم تخصيصها لتناسب الهوية الهندسية).
- **Backend**: Firebase (Firestore للبيانات، Auth للتأمين، Storage للمخططات).
- **AI Engine**: Google Gemini AI (عبر Genkit) للمساعد المحاسبي وتحليل عروض الأسعار.

---

## 📦 2. موديول المقاولات (WBS & Project Management)

هذا هو القسم الأكثر تعقيداً، حيث يعتمد على **الهياكل الشجرية اللامتناهية**.

### أ. أنواع المقاولات (Standard Workflows):
- يتم تعريف أنواع المقاولات (مثل: هيكل أسود) في الإعدادات.
- يتم بناء "شجرة مراحل" تابعة لكل نوع.
- عند إنشاء مشروع جديد، يتم "استنساخ" هذه الشجرة لتصبح هي خطة التنفيذ الفعلية للمشروع.

### ب. جدول الكميات (BOQ):
- نظام BOQ ذكي يدعم المستويات (أقسام وبنود).
- مرتبط بمكتبة بنود مرجعية موحدة.
- كل بند في الـ BOQ يمتلك (تاريخ بدء، تاريخ انتهاء، تكلفة مخططة، تكلفة فعلية).

### ج. أتمتة المستخلصات:
- **المنطق**: إذا اكتملت مرحلة في الموقع (عبر زيارة ميدانية) وكانت هذه المرحلة معرفة كـ "شرط استحقاق" في العقد، يقوم النظام آلياً بإنشاء **مسودة مستخلص** وقيد مالي بانتظار المراجعة.

---

## 💰 3. موديول المحاسبة والمالية (IFRS Compliance)

### أ. شجرة الحسابات الذكية:
- النظام يقوم بإنشاء حسابات العملاء آلياً تحت حساب "العملاء" الرئيسي بمجرد إنشاء أول عقد.
- يدعم مراكز التكلفة والربحية (كل مشروع/معاملة هو مركز ربحية مستقل).

### ب. القيود والسندات:
- **القيود الآلية**: تولد من (العقود، أذونات الاستلام، أذونات الصرف، الرواتب).
- **سندات القبض**: تقوم بتحليل بنود العقد وتوزيع المبلغ المستلم على الدفعات المستحقة آلياً في "البيان".

---

## 👥 4. الموارد البشرية (HR & Payroll)

### أ. الرواتب الذكية:
- معالجة ملفات Excel البصمة.
- نظام "الاستئذانات"؛ حيث يتجاهل محرك الرواتب خصم التأخير إذا وجد استئذان موافق عليه لهذا اليوم.
- حساب مكافأة نهاية الخدمة بدقة متناهية وفق قانون العمل الكويتي (Article 51).

## 📄 5. الأكواد المصدرية الأساسية (Core Logic Files)

### 1. تعريفات البيانات الموحدة (`src/lib/types.ts`)
تحتوي على كافة الـ Interfaces التي تضمن تناسق البيانات بين Firestore والواجهات.

```typescript
// مقتطف من أهم الأنواع
export interface ClientTransaction {
    id?: string;
    transactionNumber?: string;
    clientId: string;
    transactionType: string;
    contract?: {
        clauses: any[];
        totalAmount: number;
        financialsType?: 'fixed' | 'percentage';
    };
    stages?: any[]; // مراحل سير العمل الشجرية
}
```

### 2. محرك التنظيف والأمان المالي (`src/lib/utils.ts`)
يحتوي على دالة `cleanFirestoreData` التي تمنع الأخطاء الشائعة عند إرسال بيانات تحتوي على قيم `undefined` أو `null` غير مرغوب فيها، ودالة `numberToArabicWords` للتفقيط المالي.

### 3. نموذج قوالب العقود (`src/components/settings/contract-template-form.tsx`)
وهو النموذج الذي تم تطويره ليكون الأذكى في النظام، حيث يربط القانون بالمال بالفن.

## 💾 6. كيفية عمل النسخ الاحتياطي (Backup Strategy)

بما أن النظام يعمل على Firebase، يمكنك في أي وقت تصدير البيانات كملفات JSON عبر أداة **Firebase CLI**. أما الأكواد، فبمجرد حفظك لهذا المجلد، ستمتلك كامل المنطق البرمجي والواجهات.

**نصيحة تقنية**: يفضل دائماً مراجعة ملف `docs/backend.json` لأنه يمثل "المخطط الهيكلي" لقاعدة البيانات، وأي تغيير فيه يجب أن ينعكس على الأكواد المذكورة هنا.

طبت وطاب عملك! هذا النظام هو الآن ملكك بالكامل بكل تفاصيله الذكية.
