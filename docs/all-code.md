
# Nova ERP - الموسوعة البرمجية والتوثيقية النهائية (الإصدار اللوجستي والرقابي الموحد)

هذا المستند هو "الصندوق الأسود" لنظام Nova ERP. يحتوي على شرح تفصيلي للمنطق البرمجي، هيكل البيانات، وخرائط العلاقات المعقدة، ليكون المرجع المطلق للنسخ الاحتياطي وإعادة البناء.

---

## 🏗️ القسم 1: المعمارية الهيكلية (System Architecture)

يعتمد النظام على Next.js 14 (App Router) مع Firebase كمحرك بيانات لحظي. التفاعل بين "الميدان" (موقع المشروع) و"المكتب" (المحاسبة) يتم عبر **أتمتة القيود**.

```text
src/
├── ai/                     # محركات الذكاء الاصطناعي (Gemini 1.5 Flash)
├── app/                    # المسارات الذكية (App Router)
│   ├── dashboard/          # مركز العمليات (محاسبة، مقاولات، موارد بشرية)
├── components/             # المكونات الرسومية (UI)
│   ├── accounting/         # محرك القيود والسندات
│   ├── construction/       # محرك الـ WBS والمستخلصات والزيارات (أفقي وبطاقات وسبريد شيت)
├── services/               # العقل المنطقي (الرواتب، التواريخ، الإشعارات)
├── lib/                    # الأنماط (Types) والأدوات (Utils)
└── hooks/                  # خطافات جلب البيانات التحليلية اللحظية
```

## 🛒 القسم 11: المشتريات المباشرة والمردودات (Direct Procurement)

### 11.1 - الفاتورة المباشرة
مصممة للشركات الصغيرة لتسجيل المشتريات دون دورة مخزنية.
*   **القيد**: (Debit: Project Expense 5104 / Credit: Vendor or Cash).
*   **الربط**: تظهر التكلفة فوراً في تقرير P&L الخاص بالمشروع المختار.

### 11.2 - المردود المباشر (Direct Return)
عكس القيد المالي للمشتريات المباشرة.
*   **المنطق**: يتم ائتمان (Credit) حساب مصاريف المشروع 5104، مما يخفض التكلفة ويرفع الربحية آلياً.

## 📐 القسم 13: إدارة المواقع واللوجستيات (Field Logistics)

### 13.1 - فلسفة الربط الحصري
تم حصر **الزيارات الميدانية** لتكون مرتبطة بـ **مشاريع المقاولات** فقط لضمان سلامة حسابات التكاليف والـ P&L.

### 13.2 - محرك الإدخال الشبكي (Spreadsheet Entry)
يسمح بجدولة عدة مشاريع في شاشة واحدة لسرعة التخطيط اليومي.

### 13.3 - محرك الفرق والمقاولين (Hybrid Resources)
*   **الاختيار المتعدد**: يتم اختيار فرق العمل (Team A1, B2...) من قائمة منسدلة.
*   **تجميد البيانات (Snapshotting)**: عند حفظ الزيارة، يتم تسجيل أسماء الفرق والعمال في تلك اللحظة داخل مستند الزيارة، مما يضمن دقة السجل التاريخي.
*   **الرقابة الميدانية**: إذا كان المشروع مسنداً لمقاول، يتم تعطيل اختيار الفرق الداخلية آلياً.

### 13.4 - ذكاء التقارير التحليلية
يوفر موديول التقارير (`field-visits-reports.tsx`) تحليلاً عميقاً يعتمد على فلاتر (اليوم، الشهر، العميل، المرحلة، الفريق).

## 🛡️ القسم 15: الرقابة والتجاوز (Override Control)

### 15.1 - محرك التجاوز (Bypass Engine)
يسمح للمدير بتجاوز تنبيهات الخلل الإجرائي (مثل الشراء المباشر بدون RFQ) بوسم `isBypassed: true` لإبقاء لوحة التحكم نظيفة ومركزة على الخلل المالي الحقيقي.

---
**ملاحظة ختامية**: هذا النظام مصمم ليكون "رقيباً"؛ تم إحكام موديول العمل الميداني ليكون مرآة حقيقية للواقع اللوجستي والفني في مواقع المقاولات.
