
# Nova ERP - الموسوعة البرمجية والتوثيقية النهائية (النسخة الرقابية الشاملة)

هذا المستند هو المرجع المطلق للنظام، تم تحديثه ليشمل حلول أخطاء البناء والربط اللوجستي المتقدم والتقارير التحليلية الذكية ومعايير الثبات المالي وأتمتة السلاسل المالية.

---

## 🏗️ القسم 1: المعمارية الهيكلية

يعتمد النظام على Next.js 14 مع Firebase كقاعدة بيانات لحظية. الميزة الكبرى هي **"التكامل اللوجستي"** الذي يربط إنجاز الموقع (WBS) بالدورة المحاسبية وإدارة الموارد البشرية.

---

## 💰 القسم 9: السلاسل المالية التلقائية (The Automated Chains)

هذا هو "ذكاء" Nova ERP؛ حيث تؤدي الأفعال الميدانية والإدارية إلى نتائج مالية فورية في الميزانية العمومية:

### 9.1 - سلسلة الإنجاز الميداني ← الإيرادات
- **الحدث**: المهندس يضغط "تم الإنجاز" في الموقع لزيارة مربوطة ببند مقايسة.
- **الأتمتة**: 
    1. النظام يطابق البند مع "شروط العقد".
    2. إذا تطابق، يتم توليد **مسودة مستخلص** (Mustakhlas) آلياً.
    3. يتم إنشاء **قيد محاسبي مسودة** (مدين: العميل / دائن: إيرادات استشارات).
- **الهدف**: تقليل الفجوة الزمنية بين الإنجاز الميداني والمطالبة المالية.

### 9.2 - سلسلة اعتماد الرواتب ← المصروفات
- **الحدث**: المدير يضغط "اعتماد وصرف الرواتب" في قائمة مسيرات الرواتب.
- **الأتمتة**:
    1. حساب إجمالي الرواتب الصافية لكافة الموظفين المختارين.
    2. توليد **قيد يومية مجمع ومرحّل** (مدين: مصروف رواتب وأجور / دائن: حساب البنك).
    3. تحديث حالة كافة الكشوف إلى "مدفوع".
- **الهدف**: ضمان إثبات التكلفة البشرية في سجلات الشركة فور سدادها.

### 9.3 - سلسلة شهادات الإنجاز ← مديونية المقاولين
- **الحدث**: اعتماد شهادة إنجاز لمقاول باطن.
- **الأتمتة**:
    1. فحص وجود حساب للمقاول في شجرة الحسابات (تحت 2101)، وإنشاؤه آلياً إذا لم يوجد.
    2. توليد قيد استحقاق (مدين: تكاليف مقاولين "مركز تكلفة المشروع" / دائن: حساب المقاول الفرعي).
- **الهدف**: الرقابة الدقيقة على ديون الموردين والمقاولين وتحميلها على المشاريع الصحيحة.

## 📐 القسم 13: إدارة المواقع واللوجستيات (Field Logistics)

### 13.1 - محرك الإنجاز التراكمي الموزون
هذا هو "ميزان الصحة" للمشروع، حيث يقوم بـ:
1.  **المعالجة التراكمية**: إذا تمت زيارة نفس بند المقايسة مرتين، يقوم النظام بجمع النسب المحققة للوصول للرصيد الفعلي للبند.
2.  **الوزن المالي الفعلي**: يتم حساب نسبة إنجاز المشروع كمتوسط موزون يعتمد على القيمة المالية لكل بند (البند الأغلى سعراً يؤثر بشكل أكبر على نسبة إنجاز المشروع).

## 🛡️ القسم 16: معايير التقارير (Reporting Standards - MANDATORY)

**قاعدة ذهبية**: يُمنع منعاً باتاً بناء تقارير تحليلية أو مالية تعمل بنظام التحديث اللحظي التلقائي. 

### المعايير الإجبارية للتقارير:
1.  **زر التوليد (Generate Button)**: يجب أن يحتوي كل تقرير على زر واضح "إنشاء التقرير" أو "تحديث القراءات".
2.  **تثبيت القراءات (Stable Snapshots)**: عند الضغط على الزر، يتم جلب البيانات وتخزينها في حالة ثابتة (State). لا تتغير هذه الحالة حتى لو تغيرت البيانات الأصلية، لتمكين المدير من المراجعة والطباعة دون تشتت.
