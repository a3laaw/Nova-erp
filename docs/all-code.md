## 🏗️ القسم 1: طبقة البيانات (Data Layer)

### 1.1 - التعريفات الموحدة (src/lib/types.ts)
هذا الملف يحدد "لغة التفاهم" بين قاعدة البيانات والواجهات.

```typescript
// العميل: يمثل ملف العميل الرئيسي
export interface Client {
    id: string;           // المعرف الفريد من Firebase
    fileId: string;       // رقم الملف (مثال: 1/2024)
    nameAr: string;       // الاسم بالعربي - مطلوب للتقارير والبحث
    status: 'new' | 'contracted' | 'cancelled' | 'reContracted';
    assignedEngineer?: string; // ربط بالموظف المسؤول
    transactionCounter: number; // عداد داخلي لتوليد أرقام المعاملات
}

// المعاملة: تمثل الخدمة المقدمة للعميل (تصميم، إشراف، إلخ)
export interface ClientTransaction {
    id?: string;
    transactionNumber?: string; // مولد آلياً (مثال: CL123-TX01)
    clientId: string;
    transactionType: string;
    contract?: {
        clauses: any[];      // دفعات العقد
        totalAmount: number; // القيمة الإجمالية
        financialsType: 'fixed' | 'percentage';
    };
    stages?: any[];          // مراحل سير العمل (WBS)
}

// القيد المحاسبي: العمود الفقري للنظام المالي
export interface JournalEntry {
    id?: string;
    entryNumber: string;     // رقم تسلسلي (JV-2024-0001)
    date: any;               // تاريخ القيد
    narration: string;       // البيان
    totalDebit: number;      // إجمالي المدين (يجب أن يساوي الدائن)
    totalCredit: number;
    status: 'draft' | 'posted';
    lines: {
        accountId: string;   // الحساب المتأثر
        accountName: string;
        debit: number;
        credit: number;
        auto_profit_center?: string; // ربط المشروع/المعاملة (مركز الربحية)
        auto_resource_id?: string;   // ربط الموظف (مركز التكلفة)
    }[];
}
```

## 💰 القسم 2: المنطق المحاسبي وأتمتة القيود (Accounting Logic)

### 2.1 - شجرة الحسابات (COA)
يعتمد النظام ترميزاً خماسياً:
- **1**: الأصول (11: متداولة، 1101: نقدية).
- **2**: الالتزامات (21: متداولة، 2101: موردون).
- **3**: حقوق الملكية.
- **4**: الإيرادات.
- **5**: التكاليف والمصروفات.

### 2.2 - القيود التلقائية
1. **عند اعتماد العقد**:
   - **مدين**: حـ/ العميل (أصول)
   - **دائن**: حـ/ إيرادات استشارات (إيراد)
   - *الهدف*: إثبات المديونية الكاملة فور التوقيع.

2. **عند تسجيل سند قبض**:
   - **مدين**: حـ/ الصندوق أو البنك (أصول)
   - **دائن**: حـ/ العميل (أصول - تخفيض المديونية)

3. **عند استلام بضاعة (GRN)**:
   - **مدين**: حـ/ المخزون (أصول)
   - **دائن**: حـ/ المورد (التزامات)

## 🏗️ القسم 3: موديول المقاولات و WBS (Construction Module)

### 3.1 - محرك الـ WBS الشجري
عند إنشاء مشروع جديد، يتم استدعاء دالة `cloneWbsStructure`:
- تأخذ `constructionTypeId`.
- تقرأ المراحل من القالب المرجعي.
- تنشئ نسخة منها في `projects/{projectId}/stages`.

### 3.2 - أتمتة المستخلصات (Payment Applications)
الرابط السحري هو حقل `condition` في بنود العقد:
- المهندس يضغط "تأكيد إنجاز" في الزيارة الميدانية.
- النظام يبحث: `contract.clauses.find(c => c.condition === stageName)`.
- إذا وُجد تطابق، يولد **مستخلصاً** وقيداً محاسبياً آلياً.

- تذييل (Footer) يحتوي على العناوين.

## 🔑 القسم 7: الصلاحيات (RBAC)

الأدوار المبرمجة:
1. **Admin**: وصول كامل وحذف نهائي وتعديل إعدادات.
2. **HR**: إدارة الموظفين والرواتب فقط.
3. **Accountant**: القيود والسندات والتقارير المالية.
4. **Engineer**: إدارة المشاريع، التقارير اليومية، وتحديث المراحل.
5. **Secretary**: المواعيد والعملاء واستقبال الطلبات.
