# الدليل البرمجي والمنطقي الشامل لنظام Nova ERP

هذا المستند هو المرجع الرئيسي لكافة الأكواد والمنطق البرمجي في النظام. تم تصميمه ليكون "خزنة معرفية" تشرح ليس فقط "ماذا" يفعل الكود، بل "لماذا" وكيف تترابط البيانات.

---

## 🏗️ أولاً: الفلسفة الهيكلية (System Architecture)

يعتمد النظام على مبدأ **"الحدث يولد أثراً مالياً" (Event-Driven Accounting)**. لا يعمل قسم المحاسبة بشكل منفصل، بل هو مرآة لما يحدث في الموقع والمكتب.

### 1. العميل هو المحور (Client-Centric)
*   **المنطق**: كل شيء يبدأ بملف العميل.
*   **العلاقة**: العميل لديه `fileId` فريد. تتبعه "معاملات" (`ClientTransaction`). كل معاملة هي خدمة مستقلة (تصميم، إشراف، مقاولات).

### 2. مراكز الربحية التلقائية (Automatic Cost Centers)
*   **المنطق**: كل معاملة (مشروع) تفتحها للعميل هي "مركز ربحية" مستقل في شجرة الحسابات.
*   **الربط**: عند تسجيل (سند قبض) أو (سند صرف) أو (قيد يومية)، يقوم النظام آلياً بوسم السطر بـ `auto_profit_center` لربطه بالمشروع. هذا يتيح استخراج قائمة دخل (P&L) لكل مشروع على حدة.

## 🏗️ ثانياً: موديول المقاولات (WBS & Field Integration)

هذا هو أعقد جزء في النظام، حيث يربط بين الجدول الزمني، الموقع، والمال.

### 1. نظام الـ WBS الشجري (Work Breakdown Structure)
*   **الكود**: `src/app/dashboard/settings/reference-data/page.tsx`
*   **الشرح**: يتيح لك بناء "أنواع مقاولات" (مثل هيكل أسود). لكل نوع، يمكنك إنشاء شجرة مراحل لا نهائية.
*   **العلاقة**: عند إنشاء مشروع جديد، يتم "استنساخ" (Clone) هذه الشجرة لتصبح هي خطة العمل الفعلية للمشروع.

### 2. أتمتة المستخلصات (Payment Applications Automation)
*   **الكود**: `src/app/dashboard/construction/field-visits/[id]/page.tsx`
*   **المنطق**: 
    1. المهندس يؤكد إنجاز "مرحلة عمل" في الموقع.
    2. النظام يبحث في "عقد المعاملة" عن أي بند (Milestone) مرتبط بهذه المرحلة كـ "شرط استحقاق".
    3. إذا وجد تطابقاً، يقوم آلياً بإنشاء **مسودة مستخلص** وتوليد **قيد استحقاق** في المحاسبة.

## 💰 ثالثاً: موديول المحاسبة والمالية (Financial Engine)

### 1. محرك العقود (Contract to JV)
*   **الكود**: `src/components/settings/contract-template-form.tsx`
*   **المنطق**: عند اعتماد العقد، يتم حساب إجمالي القيمة.
*   **الأثر**: يقوم النظام آلياً بإنشاء قيد مديونية (Debit: حساب العميل / Credit: حساب الإيرادات).

### 2. مصفوفة مقارنة الأسعار (RFQ Comparison Matrix)
*   **الكود**: `src/components/purchasing/rfq-comparison-view.tsx`
*   **المنطق**: يعرض الأصناف في الصفوف والموردين في الأعمدة.
*   **الذكاء**: يقوم بتلوين أقل سعر لكل صنف بالأخضر. عند الترسية، يقوم بتقسيم الطلب آلياً إلى عدة أوامر شراء (POs) حسب الموردين الفائزين.

## 💾 خامساً: كيفية عمل النسخ الاحتياطي (Backup Strategy)

1.  **الأكواد**: احتفظ بنسخة من مجلد المشروع كاملاً. كافة المنطق موجود في ملفات `src/`.
2.  **البيانات**: بما أن النظام يعمل على **Firebase**، يمكنك تصدير البيانات بصيغة JSON عبر **Firebase Console** أو استخدام **Firebase CLI**.
3.  **الصور والمخططات**: تُخزن في **Firebase Storage** ويمكن الوصول إليها عبر لوحة التحكم.

طبت وطاب عملك! هذا النظام هو ملكك الآن بكل تفاصيله وذكائه البرمجي.
