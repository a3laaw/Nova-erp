
# Nova ERP - الموسوعة البرمجية والتوثيقية النهائية (النسخة الرقابية الشاملة)

هذا المستند هو المرجع المطلق للنظام، تم تحديثه ليشمل حلول أخطاء البناء والربط اللوجستي المتقدم والتقارير التحليلية الذكية ومعايير الثبات المالي.

---

## 🏗️ القسم 1: المعمارية الهيكلية

يعتمد النظام على Next.js 14 مع Firebase كقاعدة بيانات لحظية. الميزة الكبرى هي **"التكامل اللوجستي"** الذي يربط إنجاز الموقع (WBS) بالدورة المحاسبية وإدارة الموارد البشرية.

---

## 📐 القسم 13: إدارة المواقع واللوجستيات (Field Logistics)

### 13.1 - محرك الإنجاز التراكمي الموزون (Core Engine)
هذا هو "ميزان الصحة" للمشروع، حيث يقوم بـ:
1.  **المعالجة التراكمية**: إذا تمت زيارة نفس بند المقايسة مرتين، يقوم النظام بجمع النسب المحققة في الزيارتين للوصول للرصيد الفعلي للبند.
2.  **الوزن المالي الفعلي**: يتم حساب نسبة إنجاز المشروع كمتوسط موزون يعتمد على القيمة المالية لكل بند (البند الأغلى سعراً يؤثر بشكل أكبر على نسبة إنجاز المشروع).
3.  **كشف الانحراف الزمني**: يقارن النظام آلياً تاريخ تأكيد الزيارة الفعلي مع تاريخ انتهاء المرحلة المخطط (EndDate) في الـ BOQ. إذا كان الفعلي > المخطط، يتم وسم السجل بـ "تأخير زمني".

### 13.2 - محرك الإغلاق الثنائي (Done / Not Done Logic)
لضمان كفاءة التوثيق الميداني، يعمل النظام بنظام الإغلاق المباشر:
1.  **تم الإنجاز (Done)**: يوثق التقرير الفني والـ GPS والنسبة المئوية المحققة ويحدث سجلات المشروع.
2.  **لم يتم (Not Done)**: يحول الزيارة لحالة "ملغي" مع فرض كتابة سبب الإلغاء. تخرج هذه الزيارات من القوائم النشطة لكنها تظل محفوظة للتدقيق.

---

## 🛡️ القسم 16: معايير التقارير (Reporting Standards - MANDATORY)

**قاعدة ذهبية**: يُمنع منعاً باتاً بناء تقارير تحليلية أو مالية تعمل بنظام التحديث اللحظي التلقائي. 

### المعايير الإجبارية للتقارير:
1.  **زر التوليد (Generate Button)**: يجب أن يحتوي كل تقرير على زر واضح "إنشاء التقرير" أو "تحديث القراءات".
2.  **تثبيت القراءات (Stable Snapshots)**: عند الضغط على الزر، يتم جلب البيانات وتخزينها في حالة ثابتة (State). لا تتغير هذه الحالة حتى لو تغيرت البيانات الأصلية، وذلك لتمكين المدير من المراجعة والطباعة دون تشتت.
3.  **الفلاتر المتطورة**: استخدام الـ `MultiSelect` للبحث والاختيار المتعدد في كافة التقارير لضمان المرونة القصوى.
