

# Nova ERP - الموسوعة البرمجية والتوثيقية النهائية (الإصدار اللوجستي الموحد)

هذا المستند هو "الصندوق الأسود" لنظام Nova ERP. يحتوي على شرح تفصيلي للمنطق البرمجي، هيكل البيانات، وخرائط العلاقات المعقدة، ليكون المرجع المطلق للنسخ الاحتياطي وإعادة البناء.

---

## 🏗️ القسم 1: المعمارية الهيكلية (System Architecture)

يعتمد النظام على Next.js 14 (App Router) مع Firebase كمحرك بيانات لحظي. التفاعل بين "الميدان" (موقع المشروع) و"المكتب" (المحاسبة) يتم عبر **أتمتة القيود**.

```text
src/
├── ai/                     # محركات الذكاء الاصطناعي (Gemini 1.5 Flash)
├── app/                    # المسارات الذكية (App Router)
│   ├── dashboard/          # مركز العمليات (محاسبة، مقاولات، موارد بشرية)
├── components/             # المكونات الرسومية (UI)
│   ├── accounting/         # محرك القيود والسندات
│   ├── construction/       # محرك الـ WBS والمستخلصات والزيارات (أفقي وبطاقات وسبريد شيت)
├── services/               # العقل المنطقي (الرواتب، التواريخ، الإشعارات)
├── lib/                    # الأنماط (Types) والأدوات (Utils)
└── hooks/                  # خطافات جلب البيانات التحليلية اللحظية
```

## 📐 القسم 13: إدارة المواقع واللوجستيات (Field Logistics)

### 13.1 - فلسفة الربط الحصري
تم حصر **الزيارات الميدانية** لتكون مرتبطة بـ **مشاريع المقاولات** فقط لضمان سلامة حسابات التكاليف والـ P&L.
*   **المصدر**: يتم جلب المشاريع النشطة من كولكشن `projects`.
*   **المراحل**: يتم ربط إنجاز الزيارة ببنود المقايسة (`BOQ Items`) الخاصة بالمشروع لضمان دقة نسب الإنجاز المالي والفني.

### 13.2 - محرك الفرق الديناميكي (Team Engine)
تم استبدال الحقول النصية للفرق بنظام **Multi-Select Teams**:
- يتم اختيار الفرق من القوائم المرجعية.
- يتم عرض الفرق المختارة في جدول أفقي للمعاينة الفورية.
- **تجميد البيانات (Snapshotting)**: عند حفظ الزيارة، يتم تسجيل أسماء الفرق والعمال في تلك اللحظة داخل مستند الزيارة، مما يضمن دقة السجل التاريخي حتى لو انتقل العامل لفريق آخر مستقبلاً.

### 13.3 - ذكاء التقارير التحليلية (Logistics Analytics)
يوفر موديول التقارير (`field-visits-reports.tsx`) تحليلًا عميقًا يعتمد على:
- **تحليل التدفق الزمني**: فلترة يومية وشهرية.
- **تحليل الموارد**: تتبع أداء الفرق الفنية ومعرفة حجم تغطيتها للمواقع.
- **تحليل العملاء والمراحل**: معرفة أكثر مراحل الـ WBS استهلاكًا للوقت والزيارات.

## 🔗 القسم 14: خريطة العلاقات (The Master Map)

| الكيان المصدر | الكيان الهدف | نوع العلاقة | حقل الربط | الأثر عند الحذف |
| :--- | :--- | :--- | :--- | :--- |
| المشروع (Project) | الزيارة (Visit) | 1:N | `projectId` | منع إذا كانت الزيارة مؤكدة |
| الزيارة (Visit) | المقايسة (BOQ) | 1:1 | `plannedStageId` | تحديث نسبة الإنجاز |
| فريق العمل (Team) | الزيارة (Visit) | N:N | `teamIds` | تجميد الأسماء (Snapshot) |
| المعاملة (Tx) | القيد (JE) | 1:N | `transactionId` | منع الحذف لسلامة المراجعة |

---
**ملاحظة ختامية**: هذا النظام مصمم ليكون "رقيباً"؛ تم إحكام موديول العمل الميداني ليكون مرآة حقيقية للواقع اللوجستي والفني في مواقع المقاولات.
