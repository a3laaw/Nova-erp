# Nova ERP - الموسوعة البرمجية والتوثيقية النهائية (النسخة المصلحة والمستقرة)

هذا المستند هو المرجع المطلق للنظام، تم تحديثه ليشمل حلول أخطاء البناء والربط اللوجستي المتقدم.

---

## 🏗️ القسم 1: المعمارية الهيكلية

يعتمد النظام على Next.js 14 مع Firebase كقاعدة بيانات لحظية. الميزة الكبرى هي **"التكامل اللوجستي"** الذي يربط إنجاز الموقع (WBS) بالدورة المحاسبية.

```text
src/
├── components/
│   ├── construction/       # محرك الـ BOQ والزيارات الميدانية المتقدمة
│   ├── ui/                 # مكونات ShadCN المصلحة (Card, Table, etc)
├── hooks/
│   ├── use-subscription.ts # محرك جلب البيانات اللحظي المستقر
```

## 📐 القسم 13: إدارة المواقع واللوجستيات (Field Logistics)

### 13.1 - العرض الشبكي المكثف (Grid View)
تم تصميم ملف `field-visits-grid.tsx` ليكون شاشة العرض الرئيسية للمدير، حيث يوفر:
- **دمج البيانات**: عرض عدد الأدوار، نوع التوريد، والمرحلة الفنية في سطر واحد.
- **تتبع الـ WBS**: ربط الزيارة ببنود المقايسة آلياً.
- **إصلاح الأخطاء**: تم التأكد من استيراد كافة المكونات (Card, Building2, etc) لضمان نجاح الـ Build.

### 13.2 - محرك التجاوز الرقابي (Bypass Engine)
يسمح للمدير بتجاوز التنبيهات الإجرائية التي لا تمثل خللاً مالياً حقيقياً عبر ختم `isBypassed`.

### 13.3 - الربط المحاسبي التلقائي
- **GRN ← المورد**: تحديث أرصدة الموردين فور الاستلام.
- **إنجاز ← مستخلص**: توليد مسودة مستخلص عند إكمال بند في الموقع.

## 🛡️ القسم 15: حماية البيانات والأداء
- **الفلترة البرمجية**: حل مشكلة الفهارس (Indexes) في Firebase عبر الفلترة في جهة العميل للشاشات المعقدة.
- **منع التكرار**: تطبيق نظام `savingRef` لمنع الحفظ المزدوج في كافة النماذج.
